## Running the pipeline

This project uses `uv` for dependency management and requires Python 3.12.11.

```bash
# Activate the virtual environment
source .venv/bin/activate

# Run the pipeline with the default topic
python main.py

# Run with a custom topic (edit run_agent call in main.py or call directly)
python -c "from agents.curator_agent import run_agent; run_agent('seu tema aqui')"
```

The `.env` file at the root must contain `OPENAI_API_KEY`.

## Pipeline architecture

The pipeline is a linear workflow orchestrated by the `agno` framework:

```
TEMA → Pesquisa (Team) → Apuração (Loop) → Rankeamento → Gate Editorial → Verificação Factual → Redação → output/
```

**Step execution order** (`core/workflow_factory.py`):
1. **pesquisa** — `Team` (líder + `Pesquisador`) uses `pesquisa-noticias` skill → saves `pesquisa_<event_id>_<date>.json`
2. **apuracao_loop** — `Loop` repeats until `MIN_FONTES` (3) sources found or `MAX_TENTATIVAS_APURACAO` (3) attempts → saves `apuracao_<event_id>_<date>.md`
3. **rankeamento** — saves `ranking_*.json` with `score`, `classification`, `decision`, `priority`
4. **gate_editorial** — Python `executor` (not an agent) reads `ranking_*.json` via `deve_publicar()` and calls `stop=True` if `decision == "nao_publicar"`
5. **verificacao** — saves `verificacao_<event_id>_<date>.md`
6. **redacao** — saves `materia_<event_id>_<date>.md`

## Skills

Each skill lives in `skills/<name>/SKILL.md` and is loaded at runtime via `agno.skills.LocalSkills`. Agents call `get_skill_instructions` to read their skill before executing.

| Skill | Output format | Consumed by |
|---|---|---|
| `pesquisa-noticias` | JSON puro (7 campos) | Apurador (extrai `event_id`, `keywords`) |
| `apuracao-fontes` | Markdown (8 seções) | Rankeador, Verificador (leem do histórico) |
| `rankeamento-noticias` | JSON puro (6 campos) | `ranker_utils.deve_publicar()` (lê do arquivo) |
| `verificacao-factual` | Markdown (7 seções) | Redator (lê do histórico) |
| `redacao-jornalistica` | Markdown (artigo) | Output final |

**Important**: only the ranking JSON is read programmatically (`ranker_utils.py`). All other inter-step communication happens through conversation history, not saved files.

## Key constraints per skill output

- `pesquisa-noticias`: pure JSON, no markdown wrapping
- `apuracao-fontes`: Markdown only — no JSON blocks, no custom markers
- `rankeamento-noticias`: pure JSON; consistency rules enforced in the skill (`score ≥ 8 → "alta"`, etc.)
- `verificacao-factual`: Markdown only — no JSON blocks, no custom markers
- `redacao-jornalistica`: prose paragraphs only — no bullet points, no internal dossiê sections, 400–600 words, `## Referências` with full article URLs

## Loop condition

`core/loop_utils.contar_fontes()` counts sources in the apurador's Markdown response by scanning `## FONTES COLETADAS` for `- **Vehicle**` markers and distinct URLs. It returns the max of both counts. The loop exits when `count >= MIN_FONTES`.

## Gate editorial

`core/ranker_utils.deve_publicar()` reads the most recent `rank*.json` file from `output/`. It first attempts `json.loads()`; on `JSONDecodeError` it falls back to regex extraction of the `decision` field. Returns `True` for `"publicar"` or `"publicar_com_ressalvas"`.

## File naming conventions

| Stage | Pattern |
|---|---|
| Pesquisa | `pesquisa_<event_id>_<YYYY-MM-DD>.json` |
| Apuração | `apuracao_<event_id>_<YYYY-MM-DD>.md` |
| Rankeamento | `ranking_*.json` (must contain "rank" for gate detection) |
| Verificação | `verificacao_<event_id>_<YYYY-MM-DD>.md` |
| Matéria | `materia_<event_id>_<YYYY-MM-DD>.md` |

The `event_id` is generated by the `pesquisa-noticias` skill (e.g. `us_naval_blockade_iran_april_2026`) and should be propagated to all subsequent file names.

## Configuration

`config/settings.py` exports three constants used across the pipeline:
- `MIN_FONTES = 3` — minimum sources for apuração loop to exit
- `MAX_TENTATIVAS_APURACAO = 3` — max loop iterations
- `OPENAI_API_KEY` — loaded from `.env`

All agents use `OpenAIResponses(id="gpt-5-nano")`.
